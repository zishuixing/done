# 无人机项目流程图

这份文档用流程图把当前工程的主流程、运行流程和数据流画出来，方便后续维护时快速进入上下文。

---

## 1. 系统启动流程图

```mermaid
flowchart TD
    A[上电复位] --> B[main]
    B --> C[HAL_Init]
    C --> D[SystemClock_Config]
    D --> E[初始化 GPIO]
    E --> F[初始化 USART2]
    F --> G[初始化 TIM1~TIM4]
    G --> H[初始化 SPI1]
    H --> I[串口打印 LOVE AND LIFE]
    I --> J[done_task_Init]
    J --> K[电机驱动初始化 Int_Motor_Init]
    K --> L[无线模块初始化 Int_SI4R1_Init]
    L --> M[无线模块进入接收模式 SI24R1_RX_Mode]
    M --> N[创建 power_task]
    N --> O[创建 led_task]
    O --> P[创建 motor_task]
    P --> Q[创建 rx_task]
    Q --> R[启动 FreeRTOS 调度器]
    R --> S[系统进入多任务运行态]
```

## 3. 遥控数据接收流程图

```mermaid
flowchart TD
    A[rx_task 周期运行] --> B[调用 App_Data_Receive]
    B --> C{SI24R1_RxPacket 是否收到数据}
    C -- 否 --> D[返回 eINVALID]
    C -- 是 --> E{协议ID是否匹配}
    E -- 否 --> D
    E -- 是 --> F[计算前13字节校验和]
    F --> G{校验和是否匹配}
    G -- 否 --> D
    G -- 是 --> H[解析 pitch roll yaw throttle]
    H --> I[解析 shutdown hold_height]
    I --> J[写入 joystick_key_handle]
    J --> K[串口打印解析结果]
    K --> L[返回 eVALID]
```

## 5. 电机输出当前流程图

```mermaid
flowchart TD
    A[motor_task 启动] --> B[4个电机速度置0]
    B --> C[调用 Int_Motor_SetSpeed 下发PWM]
    C --> D[进入循环]
    D --> E[每50ms延时一次]
```

## 7. 后续建议新增的理想流程图

下面这张不是当前已实现，而是推荐你未来演进时参考的理想链路。

```mermaid
flowchart TD
    A[遥控输入] --> B[输入管理]
    B --> C[连接状态检测]
    C --> D[系统状态机]
    D --> E[控制算法]
    E --> F[电机混控]
    F --> G[PWM输出]
    D --> H[LED/蜂鸣器提示]
    D --> I[故障保护]
```

它表达的是一个更“架构化”的思路：

- 输入和状态分离
- 状态和执行分离
- 控制和驱动分离
- 显示/告警作为独立输出通道

---

## 8. 维护时怎么用这份流程图

建议你每次维护都按这个顺序看：

1. 先看“系统启动流程图”，确认初始化链路
2. 再看“任务运行关系图”，确认业务在哪些任务里跑
3. 再看“遥控数据接收流程图”，确认输入来源
4. 最后看“当前架构关系图”，判断新功能应该加在哪一层

这样你会越来越像在“设计系统”，而不是只在“找代码”。
