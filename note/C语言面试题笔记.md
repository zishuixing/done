# C语言面试题笔记

## 1. 这份笔记怎么用

这份笔记不是为了“学完整本 C 语言”，而是为了面试前几天快速复习。

使用方式建议：

1. 先把前面的高频题全部过一遍
2. 能口头讲清楚“标准回答 + 易错点”
3. 如果你是 MCU / 嵌入式方向，就重点看 `volatile`、指针、内存、结构体、位操作、编译链接
4. 面试前一天只看标题和加粗结论

这份笔记的写法尽量贴近真实面试，而不是教科书。

## 2. 高频必背题

### 2.1 `const`、`static`、`volatile` 分别有什么作用？

#### `const`

作用：

- 表示只读，不希望程序去修改这个对象
- 可以让编译器做只读限制
- 比 `#define` 更有类型信息

常见用法：

- 修饰变量
- 修饰指针
- 修饰函数参数

面试可直接说：

`const` 表示只读约束，用来限制程序修改变量。它比宏更安全，因为有类型检查，也有作用域。

#### `static`

作用：

- 修饰局部变量：生命周期变成整个程序运行期，但作用域仍在函数内
- 修饰全局变量：作用域限制在当前文件
- 修饰函数：函数只在当前文件可见

面试可直接说：

`static` 的核心是“改变生命周期”或者“限制可见范围”。局部变量加 `static` 后只初始化一次；全局变量和函数加 `static` 后只在本文件内可见。

#### `volatile`

作用：

- 告诉编译器：这个变量可能被外部改变，不能随意优化
- 每次访问都要重新读

嵌入式常见场景：

- 中断和主循环共享变量
- 硬件寄存器
- DMA 修改的数据

面试可直接说：

`volatile` 是为了防止编译器优化缓存变量值，适合中断共享变量、寄存器映射变量等场景。但它不能保证原子性，也不能保证线程安全。

易错点：

- `volatile` 不是线程安全
- `const` 不等于真的放到 ROM
- `static` 不是“全局变量”的同义词

---

### 2.2 指针和数组有什么区别？

相同点：

- 很多场景下数组名会退化成指针
- 都可以通过下标访问元素

不同点：

- 数组是一个完整对象，指针是一个变量
- `sizeof(数组)` 得到整个数组大小
- `sizeof(指针)` 得到指针本身大小
- 数组名通常不能被重新赋值
- 指针可以指向别处

面试可直接说：

数组和指针不是同一个东西。数组是一块固定大小的连续内存，指针只是保存地址的变量。数组名在表达式里很多时候会退化成指向首元素的指针，但本质上两者不同。

经典例子：

```c
int a[10];
int *p = a;

sizeof(a); // 整个数组大小
sizeof(p); // 指针大小
```

易错点：

- “数组就是指针”这句话是错的
- 函数参数里的数组本质上会退化成指针

---

### 2.3 野指针和悬空指针有什么区别？

#### 野指针

指向一个未知地址的指针。

常见来源：

- 没初始化
- 被随便赋值

#### 悬空指针

曾经指向有效内存，但那块内存已经失效了。

常见来源：

- `free` 之后没置空
- 返回了局部变量地址

面试可直接说：

野指针是“从一开始就不知道指向哪里”，悬空指针是“曾经有效但后来失效”。两者本质都是非法访问风险。

建议：

- 指针定义后先初始化
- `free(p)` 后立刻 `p = NULL`

---

### 2.4 `malloc`、`calloc`、`realloc` 的区别是什么？

#### `malloc`

- 申请指定字节数内存
- 不初始化内容

#### `calloc`

- 申请 `n * size` 字节
- 会清零

#### `realloc`

- 扩容或缩容已有内存
- 可能原地扩展，也可能搬家

面试可直接说：

`malloc` 只申请不清零，`calloc` 会清零，`realloc` 用于调整已分配内存大小，但返回地址可能变化，所以必须用临时指针接收。

易错点：

- `realloc` 失败会返回 `NULL`
- 不能直接 `p = realloc(p, new_size)` 后不判断

推荐写法：

```c
int *tmp = realloc(p, new_size);
if (tmp != NULL)
{
    p = tmp;
}
```

---

### 2.5 堆和栈的区别是什么？

#### 栈

- 系统自动管理
- 分配释放快
- 生命周期跟函数调用有关
- 空间通常较小

#### 堆

- 程序员手动管理
- 生命周期更灵活
- 需要 `malloc/free`
- 更容易内存泄漏

面试可直接说：

栈由编译器和系统自动管理，适合局部变量；堆由程序员手动申请释放，适合生命周期不固定的对象。栈快但小，堆灵活但容易出问题。

---

### 2.6 `sizeof` 和 `strlen` 的区别是什么？

#### `sizeof`

- 编译期运算符
- 求对象或类型所占字节数
- 包括数组全部空间

#### `strlen`

- 库函数
- 运行时统计字符串长度
- 不包括结尾 `\0`

例子：

```c
char a[] = "abc";
sizeof(a);   // 4
strlen(a);   // 3
```

易错点：

- `sizeof` 对指针只会得到指针大小

---

### 2.7 `struct` 和 `union` 的区别是什么？

#### `struct`

- 各成员都有独立存储空间
- 可以同时保存多个成员值

#### `union`

- 所有成员共用同一块内存
- 同一时刻通常只能可靠使用一个成员
- 大小等于最大成员的大小，再考虑对齐

面试可直接说：

`struct` 是“多个成员同时存在”，`union` 是“多个成员共享同一块内存”。`union` 常用于节省空间、做协议解析或底层字节视图切换。

---

### 2.8 `memcpy` 和 `memmove` 的区别是什么？

#### `memcpy`

- 拷贝速度快
- 源和目标不能重叠

#### `memmove`

- 可以处理重叠内存
- 更安全

面试可直接说：

`memcpy` 适合不重叠内存，`memmove` 能处理重叠区域。重叠时用 `memcpy` 会有未定义行为。

---

### 2.9 `extern` 的作用是什么？

作用：

- 声明变量或函数在别处定义
- 让多个文件共享符号

面试可直接说：

`extern` 表示“这个变量或函数不是在这里定义的，只是在这里声明一下，真正定义在别的文件里”。

注意：

- `extern int a;` 是声明
- `int a;` 在全局区一般是定义

---

### 2.10 `typedef` 和 `#define` 有什么区别？

#### `typedef`

- 给类型起别名
- 有类型语义

#### `#define`

- 预处理文本替换
- 没有类型检查

面试可直接说：

`typedef` 是类型别名，发生在编译阶段；`#define` 是预处理文本替换，发生在预处理阶段。`typedef` 更适合类型抽象，`#define` 更适合常量和条件编译。

## 3. 指针专题

### 3.1 二级指针是什么？

二级指针本质上是“指向指针的指针”。

例如：

```c
int a = 10;
int *p = &a;
int **pp = &p;
```

用途：

- 修改指针本身
- 动态二维数组
- 函数传出二级结果

面试可直接说：

一级指针是存地址，二级指针是存一级指针的地址。它常用于在函数中修改外部指针。

---

### 3.2 `int *p[10]` 和 `int (*p)[10]` 有什么区别？

#### `int *p[10]`

- 数组
- 数组里有 10 个元素
- 每个元素都是 `int *`

#### `int (*p)[10]`

- 指针
- 指向一个有 10 个 `int` 元素的数组

口诀：

- `[]` 优先级高于 `*`

---

### 3.3 函数指针是什么？

函数指针是指向函数地址的指针。

例如：

```c
int add(int a, int b)
{
    return a + b;
}

int (*fp)(int, int) = add;
```

常见场景：

- 回调函数
- 状态机表驱动
- 驱动接口抽象

嵌入式里很常见。

---

## 4. 内存与生命周期专题

### 4.1 为什么不能返回局部变量地址？

因为局部变量在函数返回后生命周期结束，栈空间可能被覆盖。

错误例子：

```c
int *func(void)
{
    int a = 10;
    return &a;
}
```

这是悬空指针。

---

### 4.2 内存泄漏是什么？

申请了内存但没有释放，而且程序后面也失去了这块内存的引用。

例子：

```c
int *p = malloc(100);
p = NULL; // 泄漏了
```

面试可直接说：

内存泄漏的本质是“内存还占着，但程序已经找不到它了”。

---

### 4.3 内存对齐是什么？

为了提高访问效率，编译器会让某些变量按特定地址边界存放。

影响：

- 结构体大小可能比成员相加更大
- 成员顺序不同会影响结构体大小

面试可直接说：

内存对齐是编译器为了效率做的空间布局规则，它会导致结构体成员之间出现填充字节。

---

## 5. 宏与预处理专题

### 5.1 宏和函数有什么区别？

#### 宏

- 预处理替换
- 没有类型检查
- 可能产生副作用
- 没有函数调用开销

#### 函数

- 编译后有真正函数实体
- 有类型检查
- 更安全

面试可直接说：

宏本质是文本替换，优点是灵活、无调用开销，缺点是没有类型检查，容易出副作用；函数更安全也更容易调试。

经典坑：

```c
#define SQUARE(x) x * x
SQUARE(1 + 2) // 结果不是 9
```

正确写法：

```c
#define SQUARE(x) ((x) * (x))
```

---

### 5.2 头文件为什么要防止重复包含？

因为同一个头文件可能被多次包含，容易导致重复定义。

常见写法：

```c
#ifndef XXX_H
#define XXX_H

...

#endif
```

## 6. 编译与链接专题

### 6.1 C 程序从源文件到可执行文件经历了哪些过程？

四步：

1. 预处理
2. 编译
3. 汇编
4. 链接

#### 预处理

- 展开宏
- 处理头文件
- 条件编译

#### 编译

- 生成汇编代码

#### 汇编

- 汇编代码变目标文件

#### 链接

- 合并多个目标文件
- 解析符号引用
- 生成可执行文件

这是高频题，尤其嵌入式很爱问。

---

### 6.2 声明和定义有什么区别？

#### 声明

- 告诉编译器这个名字存在
- 不一定分配存储空间

#### 定义

- 真正创建对象或函数实体

例子：

```c
extern int a; // 声明
int a = 10;   // 定义
```

## 7. 嵌入式 C 常考题

### 7.1 为什么中断共享变量要加 `volatile`？

因为中断可能在主循环之外修改变量，如果不加 `volatile`，编译器可能把变量缓存到寄存器里，主循环就读不到最新值。

这是 MCU 面试非常高频的问题。

---

### 7.2 什么是大小端？

#### 大端

- 高字节放低地址

#### 小端

- 低字节放低地址

判断方法常考。

示例：

```c
int is_little_endian(void)
{
    unsigned int x = 1;
    return *(unsigned char *)&x;
}
```

如果返回 1，就是小端。

---

### 7.3 位操作为什么在 MCU 里重要？

因为很多寄存器控制是按位操作的。

常见操作：

- 置位：`x |= (1 << n)`
- 清位：`x &= ~(1 << n)`
- 取位：`(x >> n) & 1`
- 翻转：`x ^= (1 << n)`

这类题常结合寄存器、GPIO、协议解析一起问。

---

### 7.4 为什么寄存器定义常用 `volatile`？

因为寄存器的值可能被硬件随时改变，不是只有 CPU 写了才会变。

例如：

```c
#define USART_SR (*(volatile unsigned int *)0x40013800)
```

如果不加 `volatile`，编译器可能优化掉本来必须保留的访问。

## 8. 常见手写代码题

### 8.1 手写 `strlen`

```c
int my_strlen(const char *s)
{
    int len = 0;
    while (*s != '\0')
    {
        len++;
        s++;
    }
    return len;
}
```

面试注意点：

- 参数为什么是 `const char *`
- 为什么不把 `\0` 计入长度

---

### 8.2 手写 `strcmp`

```c
int my_strcmp(const char *a, const char *b)
{
    while (*a && (*a == *b))
    {
        a++;
        b++;
    }
    return (unsigned char)*a - (unsigned char)*b;
}
```

---

### 8.3 手写 `memcpy`

```c
void *my_memcpy(void *dest, const void *src, int n)
{
    char *d = (char *)dest;
    const char *s = (const char *)src;

    while (n--)
    {
        *d++ = *s++;
    }

    return dest;
}
```

注意：

- 不能处理重叠区域

---

### 8.4 链表反转

```c
typedef struct Node
{
    int data;
    struct Node *next;
} Node;

Node *reverse_list(Node *head)
{
    Node *prev = NULL;
    Node *curr = head;

    while (curr != NULL)
    {
        Node *next = curr->next;
        curr->next = prev;
        prev = curr;
        curr = next;
    }

    return prev;
}
```

## 9. 常见易错题速记

### 9.1 `sizeof(数组)` 和 `sizeof(指针)` 不一样

一定要记住：

- 数组是整体
- 指针只是地址变量

### 9.2 `free` 后必须考虑置空

不然容易形成悬空指针。

### 9.3 宏一定加括号

尤其是参数和整体都要加括号。

### 9.4 `volatile` 不保证原子性

这是嵌入式面试里最爱追问的点。

### 9.5 `memcpy` 不处理重叠

重叠区域用 `memmove`。

## 10. 深度题速记

### 10.1 为什么 `volatile` 不能保证线程安全？

因为它只限制编译器优化，不保证：

- 原子性
- 执行顺序
- 互斥访问

真正线程安全还需要：

- 锁
- 临界区
- 原子操作

### 10.2 为什么结构体会有填充字节？

因为编译器要做内存对齐，提高访问效率。

### 10.3 为什么 `realloc` 要用临时变量接？

因为失败时返回 `NULL`，如果你直接赋给原指针，会把原地址丢掉，造成内存泄漏。

### 10.4 为什么数组传参会退化成指针？

因为函数参数本质上不会拷贝整个数组，而是传首元素地址。
