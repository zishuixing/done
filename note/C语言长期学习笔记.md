# C语言长期学习笔记

## 1. 这份笔记的定位

这份笔记不是“面试前一天速记版”，而是你以后长期积累 C 语言知识的主笔记。

它适合做这些事：

- 平时学习 C 语言时记录理解
- 面试题积累
- 错题复盘
- 把知识点和自己的 MCU 项目关联起来
- 逐步形成自己的嵌入式 C 知识体系

建议你把它当成一个长期更新的知识库，而不是一次性写完。

## 2. 使用原则

### 2.1 不追求一次写满

长期笔记最重要的是持续维护，不是第一天写得很完美。

你可以先写：

- 题目
- 标准回答
- 易错点

后面再慢慢补：

- 代码示例
- 项目场景
- 面试追问

### 2.2 每个知识点都尽量关联项目

你是 MCU 工程师，所以很多 C 语言知识都可以和项目结合。

比如：

- `volatile` 可以关联中断、寄存器、DMA
- 指针可以关联驱动句柄
- 结构体可以关联电机句柄、LED 句柄、协议包
- 函数指针可以关联回调、状态机
- 位操作可以关联寄存器配置

如果你能把面试题讲到项目里，面试效果会明显更好。

### 2.3 统一模板，方便复习

每道题建议都用同样格式。

这样以后你可以快速筛选：

- 哪些题没掌握
- 哪些题能讲清
- 哪些题能结合项目

## 题目：`volatile` 有什么作用？

- 分类：关键字 / 嵌入式 C
- 频率：高频
- 难度：中等
- 方向：MCU
- 状态：待复习
- 最近复习：

### 1. 标准回答

`volatile` 用来告诉编译器，这个变量可能在程序当前执行流之外被修改，所以每次访问都必须重新读取，不能被编译器优化成缓存值。

### 2. 我的理解

它的核心不是“让变量变安全”，而是“提醒编译器别偷懒”。

### 3. 易错点

`volatile` 不能保证原子性，也不能保证线程安全。

### 4. 常见追问

1. `volatile` 和 `const` 能不能一起用？
2. 中断共享变量为什么要加 `volatile`？
3. `volatile` 能不能解决多任务竞争？

### 5. 代码示例

```c
volatile uint8_t rx_flag = 0;

void USART_IRQHandler(void)
{
    rx_flag = 1;
}

int main(void)
{
    while (1)
    {
        if (rx_flag)
        {
            rx_flag = 0;
        }
    }
}
```

### 6. 嵌入式 / MCU 场景

- 中断修改、主循环读取
- 硬件寄存器
- DMA 缓冲区

### 7. 和我项目的关联

可以关联无线接收标志、串口接收标志、外设状态寄存器。

### 8. 一句话记忆

`volatile` 是防编译器优化，不是线程安全。

### 9. 下次复习要注意

要能讲清楚“为什么不能保证原子性”。

## 题目：指针和数组有什么区别？

- 分类：指针与数组
- 频率：高频
- 难度：基础
- 方向：通用 C / MCU
- 状态：待复习
- 最近复习：

### 1. 标准回答

数组是一块连续内存，是一个完整对象；指针是一个变量，保存地址。数组名在很多表达式中会退化成指向首元素的指针，但数组本身不是指针。

### 2. 我的理解

数组像“一排房子”，指针像“门牌号”。

### 3. 易错点

`sizeof(数组)` 和 `sizeof(指针)` 不一样。

### 4. 常见追问

1. 函数参数里的数组为什么会退化？
2. 数组名能不能被赋值？
3. 指针加 1 加的是几个字节？

### 5. 代码示例

```c
int a[10];
int *p = a;

sizeof(a); // 10 * sizeof(int)
sizeof(p); // 指针大小
```

### 6. 嵌入式 / MCU 场景

协议缓冲区、串口接收缓冲区、无线数据包通常都是数组。

### 7. 和我项目的关联

`rxbufer[32]` 是数组，传给函数时会退化成指针。

### 8. 一句话记忆

数组是内存本体，指针是地址变量。

### 9. 下次复习要注意

重点看函数传参时数组退化的问题。

## 题目：`static` 有哪些作用？

- 分类：关键字
- 频率：高频
- 难度：基础
- 方向：通用 C / 嵌入式 C
- 状态：待复习
- 最近复习：

### 1. 标准回答

`static` 修饰局部变量时，会让变量生命周期变成整个程序运行期，但作用域仍在函数内。修饰全局变量或函数时，会限制它们只在当前文件内可见。

### 2. 我的理解

`static` 要么让变量“活得更久”，要么让符号“只在本文件可见”。

### 3. 易错点

局部静态变量只初始化一次。

### 4. 常见追问

1. `static` 局部变量存在哪里？
2. `static` 全局变量和普通全局变量区别？
3. 为什么驱动文件里的私有函数常加 `static`？

### 5. 代码示例

```c
void count(void)
{
    static int n = 0;
    n++;
}
```

### 6. 嵌入式 / MCU 场景

驱动文件内部函数可以加 `static`，避免暴露给其他模块。

### 7. 和我项目的关联

`SPI_RW()` 这种只给 `Int_SI4R1.c` 内部用的底层函数，就适合用 `static`。

### 8. 一句话记忆

`static` 控制生命周期和可见性。

### 9. 下次复习要注意

区分局部 `static` 和全局 `static`。
