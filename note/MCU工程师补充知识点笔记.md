# MCU工程师补充知识点笔记

## 1. 文档定位

这份笔记是对你已经整理好的：

- C语言
- FreeRTOS
- STM32

这三类主干内容的补充。

如果目标是面试 MCU 工程师，那么除了这三块之外，还需要继续整理一些“工程味更强”的知识点。  
这些内容往往决定你能不能从“会写代码”升级到“会做板级系统、会调试、会和硬件协作”的水平。

这份文档的目标是：

1. 帮你建立 MCU 面试的完整知识地图
2. 告诉你每一块为什么重要
3. 告诉你每一块具体该整理什么
4. 帮你安排后续整理顺序

---

## 2. 数字电路基础

这是 MCU 工程师非常容易被问到，但很多人准备不系统的一块。

### 为什么重要

因为 MCU 工程师不是只面对软件，你写的 GPIO、电平、中断、通信，最后都落在实际硬件上。

如果数字电路基础不清楚，很容易出现：

- GPIO 会配，但不知道为什么这么配
- 知道开漏和推挽名字，但讲不出场景
- 知道上拉下拉，但讲不清作用

### 建议整理的内容

- 高电平 / 低电平
- 上拉 / 下拉
- 推挽输出 / 开漏输出
- 三态输出
- 施密特触发基础
- 按键消抖
- 电平转换
- MOS 管和三极管基础
- 二极管作用
- RC 延时与滤波
- 线与、线或概念

### 面试常问题

- 推挽和开漏有什么区别？
- 为什么 I2C 要开漏？
- 上拉下拉的作用是什么？
- 按键为什么要消抖？
- 为什么某些引脚悬空会出问题？

### 你可以怎么记

建议以后单独做一本：

- `数字电路面试题笔记.md`

---

## 3. 模拟电路与电源基础

这块对 MCU 工程师越来越重要，尤其是你这种已经做过真实板级项目的人。

### 为什么重要

很多项目问题本质上不是代码错，而是：

- 供电不稳
- 地弹噪声
- ADC 采样不准
- 无线模块瞬时掉压
- 电容布局不合理

你自己的 SI24R1 电池供电问题就是一个很典型的例子。

### 建议整理的内容

- LDO 和 DCDC 区别
- Boost / Buck 基础
- 电源纹波
- 去耦电容 / 滤波电容
- 旁路电容
- 电池供电基础
- 模拟地 / 数字地
- ADC 参考电压
- 采样噪声来源
- 低功耗设计
- 掉电保存
- 上电时序

### 面试常问题

- LDO 和 DCDC 有什么区别？
- 为什么 ADC 采样会抖动？
- 去耦电容有什么作用？
- 电池供电和 USB 供电有什么差异？
- 低功耗设计一般怎么做？

### 你项目里的可讲点

- SI24R1 电池供电不稳定
- 初始化前需要延时等待 boost 稳定
- 需要示波器观察电压爬升波形

这个部分非常值得以后拿去当项目亮点讲。

## 4. 常见通信协议

虽然你已经在 STM32 笔记里整理了一些外设知识，但建议把“协议”再单独整理一次。

### 为什么重要

因为 MCU 面试里常常不是问你：

- “会不会配 SPI”

而是问你：

- “SPI 和 I2C 怎么选？”
- “UART 为什么容易出乱码？”
- “CAN 为什么适合工业控制？”

也就是说，协议层的理解要比单纯 HAL 初始化更高一层。

### 建议整理的内容

- UART
- SPI
- I2C
- CAN
- RS485
- USB 基础
- PWM
- ADC / DAC
- 1-Wire
- 红外协议基础

### 每种协议建议整理 6 项

1. 原理
2. 线数
3. 时序特点
4. 优缺点
5. 应用场景
6. 常见故障排查

### 面试常问题

- SPI 和 I2C 区别？
- UART 为什么不需要时钟线？
- RS485 和 UART 是什么关系？
- CAN 为什么抗干扰强？
- 为什么有的模块更适合 SPI，不适合 I2C？

---

## 5. 中断与实时系统设计

这块和 FreeRTOS 有关系，但建议你单独整理成“实时系统视角”。

### 为什么重要

很多 MCU 面试官会问：

- 中断里能不能做这个？
- 主循环和中断怎么同步？
- 周期任务怎么设计？
- 任务和中断怎么交互？

这些问题不只是 RTOS 问题，而是整个嵌入式实时设计问题。

### 建议整理的内容

- 中断执行原则
- 中断里不能做什么
- 中断共享变量
- 中断和主循环交互
- 中断和任务交互
- 周期任务设计
- 超时机制
- 软件定时器 / 硬件定时器
- 实时性怎么理解
- 去抖动和滤波
- 时间基准设计

### 面试常问题

- 中断里为什么不能放太多代码？
- 为什么中断共享变量要加 `volatile`？
- 周期任务为什么更适合 `xTaskDelayUntil()`？
- 中断和任务之间怎么传数据？

## 6. 驱动设计与软件分层

这块是你现在最值得提升的内容之一。

### 为什么重要

初级工程师常见问题是：

- 会写功能
- 但不会分层
- 不清楚驱动层和应用层边界

而 MCU 面试中，如果你能讲清：

- 驱动怎么封装
- 句柄为什么这样设计
- 模块之间怎么协作

会很加分。

### 建议整理的内容

- 驱动层 / 业务层 / 应用层
- 接口设计
- 句柄设计
- 初始化函数设计
- 回调函数
- 模块解耦
- 状态机设计
- 配置与运行分离
- 协议解析层和驱动层分离

### 你项目里的可讲点

- `Int_SI4R1`
- `Int_Motor`
- `Int_Led`
- `App_Data`
- `done_task`

这几个模块非常适合拿来总结“分层设计”。

## 7. 调试与排错方法

这部分在 MCU 面试里非常重要，甚至有时候比背知识点更重要。

### 为什么重要

面试官会更关心：

- 出问题时你怎么查

而不是只关心：

- 这个 API 会不会背

### 建议整理的内容

- 外设不工作怎么查
- 下载失败怎么查
- 串口乱码怎么查
- SPI 收发异常怎么查
- ADC 不准怎么查
- 系统死机怎么查
- HardFault 怎么查
- RTOS 卡死怎么查
- 示波器怎么用
- 逻辑分析仪怎么用
- GPIO 翻转调试法
- 日志调试法

### 面试常问题

- 如果串口不通你怎么排查？
- 如果程序卡死你先看什么？
- 你平时怎么定位硬件问题？
- 为什么最后还是要看波形？

### 强烈建议

这一块以后完全可以单独整理成一本：

- `MCU调试与排错面试题笔记.md`

这对面试非常有帮助。

## 8. 项目经验提炼

这不是传统意义上的“知识点”，但对面试结果影响非常大。

### 为什么重要

很多 MCU 面试到后面都会转向：

- 项目里你做了什么
- 遇到什么问题
- 你怎么解决

如果这部分讲不好，前面知识点再会，也会显得不够落地。

### 建议你整理的内容

- 项目背景
- 板子功能
- 你负责的模块
- 用了哪些外设
- 用了哪些协议
- 用了哪些 RTOS 机制
- 遇到的核心问题
- 你怎么定位
- 你怎么解决
- 如果重来一次你怎么优化

### 你现在已有的好素材

- 无人机接收执行端
- 遥控器端
- SI24R1 无线通信
- 电池供电稳定性问题
- FreeRTOS 任务拆分
- 状态机雏形

这些都非常适合转成项目面试材料。

## 9. 数据结构与算法基础

MCU 岗不会像算法岗那么卷，但基础还是要有。

### 建议整理的内容

- 链表
- 队列
- 栈
- 环形缓冲区
- 查找
- 排序基础
- 位运算题
- 字符串处理

### 为什么 MCU 也要整理

因为很多嵌入式代码本质上就在用这些结构：

- 环形缓冲区做串口接收
- 队列做任务通信
- 链表在 RTOS 内核里很常见

### 面试常问题

- 环形队列怎么实现？
- 链表反转会不会？
- 你怎么判断大小端？
- `strlen` / `memcpy` 会不会手写？

## 12. MCU 面试里的软技能题

这个很多人忽略，但面试经常会问。

### 建议准备

- 你怎么学习一个新芯片？
- 遇到不会的问题怎么办？
- 你怎么和硬件工程师协作？
- 你是怎么定位难 bug 的？
- 为什么用 RTOS / 为什么不用 RTOS？
- 你项目中最困难的点是什么？
- 你未来想往哪个方向发展？

### 准备建议

这类问题不要答空话，尽量结合你真实项目经历。

比如：

- “我会先看参考手册、原理图，再搭最小验证”
- “我会先把问题缩小到时钟、GPIO、外设、波形四层”

## 15. 一句话结论

如果要面试 MCU 工程师，除了 C / FreeRTOS / STM32 之外，你最该继续整理的是：

- 数字电路基础
- 电源与低功耗
- 通信协议
- 调试与排错
- 驱动分层与项目经验

因为真正面试里，决定你像不像“工程师”的，往往不是 API 会不会背，而是：

**你能不能把软件、硬件、调试、架构和项目经验串成一个完整系统。**
